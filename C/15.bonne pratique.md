<span style="color: #3498db">15. bonnes pratiques</span>

Pour écrire du code **lisible, maintenable et sûr**, il est important de suivre certaines pratiques standards.

---

### **1. Nommer clairement les variables**

* Choisir des noms **significatifs** :

```c
int age;           
float tauxConversion;
```

* Éviter les noms trop courts ou ambiguës comme `a`, `x`, `tmp` sauf dans de petites boucles temporaires.
* **Ne pas utiliser d’accents, caractères spéciaux ou espaces** dans les noms de variables.
* Éviter les majuscules au début si vous utilisez le style `snake_case`, ou rester cohérent si vous utilisez `camelCase`.

```c
int taille_max;     // correct
int TailleMax;      // correct si cohérent avec le style camelCase
int tailleMax;      // correct en camelCase
int tailleMax!;     // incorrect
int élève;          // incorrect (accent)
```

---

### **2. Commenter le code**

* Expliquer **le rôle d’une fonction** ou d’un bloc complexe :

```c
// Cette fonction calcule la surface d'un cercle
float surfaceCercle(float rayon) {
    return 3.14159 * rayon * rayon;
}
```

* Ne pas commenter **l’évidence** (ex : `i++` dans une boucle simple)
* Les commentaires servent surtout à **clarifier la logique**

---

### **3. Éviter les répétitions (DRY – Don’t Repeat Yourself)**

* Factoriser le code répétitif dans des fonctions :

```c
void afficherMessage(char *msg) {
    printf("Message : %s\n", msg);
}

// Utilisation
afficherMessage("Bonjour");
afficherMessage("Au revoir");
```

* Permet de **réduire les erreurs et faciliter les modifications**

---

### **4. Tester régulièrement**

* Compiler et exécuter **souvent** pour détecter les erreurs tôt
* Vérifier les entrées utilisateur et les retours de fonctions :

```c
if (malloc(10 * sizeof(int)) == NULL) {
    printf("Erreur d'allocation\n");
}
```

* Utiliser des petits programmes de test pour **chaque fonction critique**

---

### **5. Autres bonnes pratiques**

* Utiliser **des espaces et l’indentation** pour améliorer la lisibilité
* Respecter une **convention de nommage cohérente** (camelCase, snake_case…)
* Fermer **tous les fichiers ouverts** et libérer la **mémoire dynamique**
* Éviter les **variables globales** inutiles

---

### **Résumé rapide**

1. Variables avec noms clairs, **sans accents ni caractères spéciaux**, cohérents pour les majuscules
2. Commenter le code utile
3. Factoriser pour éviter les répétitions
4. Tester régulièrement et vérifier les erreurs
5. Indentation et conventions cohérentes
