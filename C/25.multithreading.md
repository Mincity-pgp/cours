<span style="color: #3498db">25. multithreading et parallélisme</span>

Le multithreading permet de **faire plusieurs tâches en même temps**, ce qui accélère le programme sur les processeurs multicœurs.

---

### **1. Concepts de base**

* **Thread** = fil d’exécution indépendant dans un programme
* Plusieurs threads peuvent **partager la mémoire**, donc attention aux conflits
* **Parallélisme** = exécution simultanée sur plusieurs cœurs
* **Concurrence** = gestion de plusieurs tâches en même temps, pas forcément en simultané

---

### **2. Créer un thread avec pthread (Linux)**

```c id="f1k9xz"
#include <pthread.h>
#include <stdio.h>

void* tache(void* arg) {
    int n = *(int*)arg;
    printf("Thread numéro %d\n", n);
    return NULL;
}

int main() {
    pthread_t thread1, thread2;
    int num1 = 1, num2 = 2;

    pthread_create(&thread1, NULL, tache, &num1);
    pthread_create(&thread2, NULL, tache, &num2);

    pthread_join(thread1, NULL); // attendre la fin du thread
    pthread_join(thread2, NULL);

    return 0;
}
```

* `pthread_create` → crée un thread
* `pthread_join` → attend qu’un thread se termine
* Chaque thread reçoit un **argument `void*`** → typiquement un pointeur vers les données

---

### **3. Synchronisation des threads**

* Quand plusieurs threads partagent des données, il faut éviter les **conflits (race conditions)**
* Utiliser **mutex** (verrou) :

```c id="g2l8pm"
pthread_mutex_t lock = PTHREAD_MUTEX_INITIALIZER;
int compteur = 0;

void* tache(void* arg) {
    pthread_mutex_lock(&lock);
    compteur++;
    pthread_mutex_unlock(&lock);
    return NULL;
}
```

* `pthread_mutex_lock` → bloque l’accès aux autres threads
* `pthread_mutex_unlock` → libère le verrou

---

### **4. Threads et performance**

* Diviser le travail en **plusieurs threads** → gain de temps si le travail est lourd
* Trop de threads → surcharge et ralentissement → **adapter au nombre de cœurs**

```c id="h3m9qn"
// Exemple simple : utiliser 4 threads pour calculer des sommes partielles
int n = 1000000;
int sum[4] = {0};
pthread_t threads[4];

// chaque thread calcule 1/4 des nombres
```

* **Conseil** : mesurer toujours le gain, car le parallélisme a un **coût de création et synchronisation**

---

### **5. Parallélisme avec OpenMP (plus simple)**

* OpenMP permet de **paralléliser facilement les boucles** :

```c id="i4n0ro"
#include <omp.h>
#include <stdio.h>

int main() {
    int somme = 0;
    #pragma omp parallel for reduction(+:somme)
    for (int i = 1; i <= 1000; i++) {
        somme += i;
    }
    printf("Somme = %d\n", somme);
    return 0;
}
```

* `#pragma omp parallel for` → exécute la boucle sur plusieurs threads

* `reduction(+:somme)` → combine correctement les résultats de chaque thread

* Très utile pour les **calculs lourds sur de grands tableaux**

---

### **6. Bonnes pratiques**

1. Ne paralléliser que les **tâches lourdes**
2. Minimiser l’accès simultané à la mémoire partagée
3. Toujours **tester pour race conditions et deadlocks**
4. Limiter le nombre de threads au **nombre de cœurs disponibles**
5. Profiler pour **mesurer l’amélioration réelle**
