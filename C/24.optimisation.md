<span style="color: #3498db">24. optimisation du code en C</span>

L’optimisation consiste à **réduire le temps d’exécution et la consommation mémoire** sans sacrifier la clarté et la sécurité du code.

---

### **1. Optimisation des boucles**

* Éviter les calculs **inutiles dans les boucles**

```c id="f1k9xz"
// Mauvais
for (int i = 0; i < strlen(str); i++) {
    printf("%c", str[i]);
}

// Bon
int len = strlen(str);
for (int i = 0; i < len; i++) {
    printf("%c", str[i]);
}
```

* Calculer une fois ce qui peut être calculé hors de la boucle
* Préférer les **boucles `for` simples** et éviter les boucles imbriquées inutiles

---

### **2. Choix des types de variables**

* Utiliser le type **adapté** pour économiser de la mémoire et accélérer :

| Type             | Usage optimal                             |
| ---------------- | ----------------------------------------- |
| `int`            | entiers généraux                          |
| `short` / `char` | pour petites valeurs, économise mémoire   |
| `float`          | calculs rapides sur nombres réels simples |
| `double`         | précision, mais plus lent que float       |

* Les **variables locales** sont plus rapides que globales (stockées dans le **stack**)

---

### **3. Éviter les allocations dynamiques inutiles**

* `malloc` / `free` sont **coûteux** en temps d’exécution
* Préférer **tableaux statiques** si la taille est connue

```c id="g2l8pm"
int tab[100]; // rapide
int *tab = malloc(100 * sizeof(int)); // plus lent, plus fragile
```

* Toujours libérer la mémoire correctement pour éviter les fuites

---

### **4. Optimisation des conditions**

* Éviter les conditions **complexes et répétitives**

```c id="h3m9qn"
// Mauvais
if (a > 10 && a < 20 && a != 15) { ... }

// Bon
int cond = (a > 10 && a < 20 && a != 15);
if (cond) { ... }
```

* Placer les **conditions les plus fréquentes en premier** pour réduire les tests

---

### **5. Optimisation des fonctions**

* **Passage par référence** pour les gros objets (tableaux, structures)
* **Inline** pour les petites fonctions répétées :

```c id="i4n0ro"
inline int carre(int x) {
    return x*x;
}
```

* Évite l’**appel de fonction classique**, réduit le temps d’exécution

* **Minimiser les appels de fonctions coûteuses** dans les boucles

---

### **6. Optimisation des chaînes et tableaux**

* Utiliser **pointeurs** pour parcourir les tableaux rapidement

```c id="j5o1sp"
char str[] = "Bonjour";
for (char *p = str; *p != '\0'; p++) {
    printf("%c", *p);
}
```

* Éviter de recalculer `strlen` à chaque itération
* Utiliser **tableaux contigus en mémoire** pour les grands tableaux multidimensionnels

---

### **7. Optimisation des opérations mathématiques**

* Préférer **opérations simples** à des fonctions complexes

```c id="k6p2tq"
// Mauvais
double r = sqrt(pow(x, 2) + pow(y, 2));

// Bon
double r = sqrt(x*x + y*y);
```

* Éviter les conversions inutiles (`int → float → int`)
* Utiliser les **constantes** et `#define` pour éviter les calculs répétitifs

---

### **8. Optimisation de l’entrée/sortie (I/O)**

* `printf` et `scanf` sont **lents**, surtout dans les boucles
* Préférer **bufferiser l’I/O** :

```c id="l7q3vy"
char buffer[1024];
fgets(buffer, sizeof(buffer), stdin);
```

* Pour fichiers, lire/écrire **blocs entiers** plutôt qu’un caractère à la fois

---

### **9. Optimisation de la mémoire cache**

* Accéder aux **tableaux dans l’ordre de stockage en mémoire**
* Favoriser les **tableaux contigus** pour éviter les cache misses
* Les boucles imbriquées doivent parcourir les **lignes avant les colonnes** pour les tableaux 2D

```c id="m8r4wx"
// Correct si tableau 2D stocké en ligne-major
for (int i = 0; i < lignes; i++)
    for (int j = 0; j < colonnes; j++)
        matrice[i][j] = i+j;
```

---

### **10. Utiliser les options du compilateur**

* GCC : `-O1`, `-O2`, `-O3` → niveaux d’optimisation
* `-march=native` → utilise les instructions spécifiques à ton CPU
* `-flto` → Link Time Optimization, optimise tout le programme

```bash
gcc -O2 -march=native -flto programme.c -o programme
```

* Les optimisations doivent être **testées**, car certaines peuvent changer le comportement si le code est fragile

---

### **11. Profiler et mesurer**

* Avant d’optimiser, **mesurer le temps d’exécution** avec `clock()`

```c id="n9s5yz"
#include <time.h>
clock_t start = clock();
// code à tester
clock_t end = clock();
double temps = (double)(end - start) / CLOCKS_PER_SEC;
printf("Temps = %.6f s\n", temps);
```

* Identifier les **parties lentes** → optimiser là où ça compte

---

### **12. Bonnes pratiques générales**

1. **Lire et écrire clair** → code lisible = plus facile à optimiser
2. Réutiliser les calculs fréquents
3. Préférer **variables locales et constantes**
4. Éviter les allocations répétitives
5. Optimiser **après avoir mesuré** → ne pas deviner
6. Profiler régulièrement sur différents volumes de données
