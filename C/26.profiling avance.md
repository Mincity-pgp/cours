<span style="color: #3498db">26. profiling avancé et optimisation finale</span>

Le **profiling** consiste à mesurer précisément **où le code consomme du temps et de la mémoire**, puis appliquer des optimisations ciblées.

---

### **1. Pourquoi profiler ?**

* Identifier les **goulots d’étranglement**
* Optimiser uniquement ce qui **ralentit vraiment** le programme
* Éviter d’optimiser du code qui n’a **pas d’impact significatif**

---

### **2. Outils de profiling**

* **Linux / GCC** : `gprof`

```bash
gcc -pg programme.c -o programme
./programme
gprof programme gmon.out > analyse.txt
```

* **Valgrind** : détecte fuites mémoire et opérations coûteuses

```bash
valgrind --tool=callgrind ./programme
```

* **Perf** : analyse les performances CPU et cache

```bash
perf stat ./programme
```

* **Visual Studio Profiler** (Windows) pour mesurer CPU, mémoire et I/O

---

### **3. Mesurer le temps d’exécution dans le code**

```c id="f1k9xz"
#include <time.h>
#include <stdio.h>

int main() {
    clock_t start = clock();
    // code à profiler
    clock_t end = clock();
    double temps = (double)(end - start) / CLOCKS_PER_SEC;
    printf("Temps = %.6f s\n", temps);
}
```

* Permet de **tester différentes versions du code**

---

### **4. Optimisation finale : bonnes pratiques**

1. **Éliminer les calculs redondants**

   * Stocker les résultats intermédiaires plutôt que recalculer

2. **Optimiser les boucles**

   * Préférer les boucles simples et éviter les opérations complexes dans l’itération
   * Utiliser des pointeurs pour parcourir les tableaux

3. **Réduire les allocations dynamiques**

   * Préférer des **tableaux statiques ou réutiliser la mémoire**

4. **Minimiser les accès concurrents** (si multithreading)

   * Limiter les mutex et locks au strict nécessaire

5. **Utiliser des types adaptés**

   * `int`, `short`, `float` selon les besoins pour réduire l’utilisation CPU et mémoire

6. **I/O efficace**

   * Bufferiser lecture/écriture
   * Éviter d’utiliser `printf` dans des boucles critiques

7. **Compiler avec optimisation**

```bash
gcc -O3 -march=native -flto programme.c -o programme
```

* `-O3` → optimisation maximale
* `-flto` → optimisation à la liaison (Link Time Optimization)

8. **Profiler avant et après chaque changement**

   * Vérifier que l’optimisation apporte vraiment un gain

---

### **5. Astuces avancées**

* **Loop unrolling** : dérouler les petites boucles pour réduire les tests de condition
* **Préfetching** : anticiper la lecture mémoire pour profiter du cache
* **Constantes et `inline`** : réduire les appels de fonctions répétitifs
* **Parallélisation** : OpenMP ou pthread pour les tâches lourdes sur plusieurs cœurs

---

### **6. Résumé**

* **Profiler → Identifier → Optimiser**
* Optimiser uniquement **les parties critiques**
* Combiner : **types adaptés, boucles efficaces, mémoire optimisée, multithreading**
* Toujours **tester et mesurer** pour éviter les régressions
