<span style="color: #3498db">23. gestion des erreurs et assertions</span>

Pour écrire du code fiable, il est essentiel de **détecter, gérer et prévenir les erreurs**. Le langage C fournit des mécanismes pour gérer les erreurs à l’exécution et vérifier les hypothèses dans le code.

---

### **1. Vérification des retours de fonctions**

* De nombreuses fonctions retournent une **valeur spéciale** en cas d’erreur.
* Exemple : ouverture de fichier

```c id="f1k9xz"
#include <stdio.h>

FILE *f = fopen("exemple.txt", "r");
if (f == NULL) {
    perror("Erreur ouverture fichier");
    return 1; // quitter le programme
}
fclose(f);
```

* `perror` affiche un message lié à l’erreur système
* Toujours vérifier les **retours de malloc, fopen, scanf…**

---

### **2. errno et gestion des erreurs système**

```c id="g2l8pm"
#include <errno.h>
#include <string.h>

FILE *f = fopen("inexistant.txt", "r");
if (!f) {
    printf("Erreur %d : %s\n", errno, strerror(errno));
}
```

* `errno` → code de l’erreur
* `strerror(errno)` → description lisible

---

### **3. Assertions**

* Vérifier les hypothèses dans le code avec `assert`

```c id="h3m9qn"
#include <assert.h>

int diviser(int a, int b) {
    assert(b != 0); // le programme s'arrête si b==0
    return a / b;
}

int main() {
    int x = diviser(10, 2); // OK
    int y = diviser(10, 0); // Erreur → arrêt du programme
}
```

* `assert(condition)` → stoppe le programme si la condition est fausse
* Utile pour **déboguer et tester**

---

### **4. Gestion d’erreurs personnalisée**

* On peut créer des fonctions pour gérer les erreurs et afficher un message clair

```c id="i4n0ro"
void erreur(const char *msg) {
    fprintf(stderr, "Erreur : %s\n", msg);
    exit(1); // quitter le programme
}

int main() {
    int *tab = malloc(1000 * sizeof(int));
    if (!tab) erreur("allocation mémoire échouée");
}
```

* `stderr` → flux d’erreurs
* `exit(code)` → quitter le programme immédiatement

---

### **5. Bonnes pratiques**

1. Vérifier **tous les retours de fonctions critiques** (`malloc`, `fopen`, `scanf`, etc.)
2. Utiliser `assert` pour valider les hypothèses importantes
3. Fournir des **messages d’erreur clairs**
4. Libérer toutes les ressources avant de quitter (`free`, `fclose`)
5. Ne jamais ignorer les erreurs système
